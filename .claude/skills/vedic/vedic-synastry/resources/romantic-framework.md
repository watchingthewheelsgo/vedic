# 情感框架 (romantic-framework)

> 适用：恋爱、婚姻、亲密伴侣。默认框架。
> 进入：Layer 0.5 平扫后 intake 选「情感」。
> **性别处理（分层，传统/非传统都不打折）**：
> - **传统异性恋 + 已知性别** → 正式启用性别 Karaka：女方盘看 Jupiter(夫)、男方盘看 Venus(妻)，配合 7宫/DK/UL，吃满传统精度。
> - **非二元 / 同性 / 未提供性别** → A/B 方向 + DK + Venus（中性），不预设异性、不套男女角色。
> 两层叠加：性别信息在就用，不在就中性兜底。

---

## Layer 0 双盘资格：读什么

- **D1**：5宫(恋爱)、7宫+7宫主(婚姻)、Venus(爱与吸引)、Moon(情绪需求)、DK(配偶指示)、UL(婚姻来源)
- **D9**：上升、7宫+7宫主、Venus、DK（婚姻深层品质）
- **Dasha**：当前/未来是否激活 5/7/8/11/12宫、Venus、DK、UL
- 承载卡四问：能提供的情感供给 / 需要什么 / 压力下如何失衡 / 当前能否进入承诺

## Layer 2 重点叠盘信号

- 吸引与亲密：Venus-Mars、Moon-Venus、Venus-Jupiter；Mars 触对方 5/7/8宫
- 长期承载：Saturn 触 Moon/Venus/7宫/UL；Jupiter 护 7宫/UL；双方 D9 7宫
- 理想化/黏着：Rahu-Ketu 轴触 Moon/Venus/Lagna/7宫轴
- 每个风险配修复：Jupiter 视相、宫主强、Dasha 支持

## 六维矩阵侧重

高权重：**吸引与亲密 / 长期承载 / 情绪安全**。六维全用。

## 本框架关系类型词

高吸引高承载 / 高吸引低承载 / 低戏剧高承载 / 成长催化型 / 节奏错位型 / 高消耗修复不足

## 关系阶段/形态适配（可选；未指定 → 通用读法）

- **暗恋/单向 crush**：重点看方向不对称（A→B 强、B→A 弱 = 单向投射）。⚠️ 诚实边界：只能看你的投射强度 + 对方点位是否被触动，**不能断言对方的感受**。
- **暧昧/situationship**：聚焦"为什么卡着不升级"——吸引有，但 7宫/UL/Saturn 承载弱 + 时机错位。诚实说能否升级，不美化成"快在一起了"。
- **短期**：高吸引低承载，明说"激情型，长期需额外条件"。
- **恋爱中**：标准 romantic 读法。
- **已婚**：重 7宫/UL/D9 承载 + 长期 Dasha。
- **分合纠缠**：Rahu-Ketu/Saturn 业力 + 8/12宫，看是否反复。
